「きさらぎ駅」へのオマージュ。深夜のオカルト板の“神スレ”に、住人の一人として
書き込みながら立ち会う一本道ホラー体験。**単一ファイル（index.html）で完結**。

公開: https://play.bubblegumgameboy.com/

## 1. ホスティング / ドメイン / HTTPS（※将来の確認用・事前情報）

- ホスト: **GitHub Pages**。リポジトリ `bubblegumgameboy/2chgame`、**`main` ブランチの root** を配信。
- 独自ドメイン: リポジトリ root の **`CNAME` ファイル**＝`play.bubblegumgameboy.com`。
- DNS（ドメイン管理側のCNAMEレコード）:
  - `play.bubblegumgameboy.com` → `bubblegumgameboy.github.io`
  - `www.bubblegumgameboy.com`  → `bubblegumgameboy.github.io`
  - ※apex（`bubblegumgameboy.com`）を使う場合はCNAME不可。GitHubのIP宛Aレコードが必要。
- **HTTPS**: GitHub → Settings → Pages →「**Enforce HTTPS**」を有効化。
  - 「安全ではありません」が出る時の確認順:
    1. `https://` で開いているか（`http://` や `file://` はNG）
    2. Settings→Pages の Custom domain が `play.bubblegumgameboy.com`／緑チェックか
    3. 「Enforce HTTPS」がオンか（証明書発行直後は数分〜1時間待ち）
- 構成ファイル: `index.html`（単一）, `CNAME`, `station.png`, `tunnel.png`。

## 2. 広告（忍者AdMax）

- **PC（横1100px以上）**: 右端固定の縦バナー **160×600**（`#ad-side`）
- **スマホ/タブレット（1100px未満）**: タイトル下に **320×100**（`#ad-mobile`）
- 互いに排他＝どの端末でも広告は1個だけ。
- 方式: **HTMLの静的スロットにタグを直接埋め込み**（iframe隔離はしない）。
  iframe(srcdoc)に入れると実ページのリファラが無く**配信されない**ため。
- 出ない時の切り分け: `file://`で開いてない / アドブロック / 在庫切れ / HTTPS未確定 / 横1100px未満でPC枠は非表示。
- タグは `index.html` 内に直書き（差し替えは該当 `<script src=...>` を置換）。

## 3. アーキテクチャ（重要）

- **完全に単一ファイル。ビルド不要。** 開けば動く。
- 物語は**データ駆動**: `const SCRIPT = [ {posts:[...], think:"..."}, ... , {end:true, endNote:"..."} ]`
  - `posts[]`: 自動で流れるレス。`N()`=名無し（うしろの名無しさん）/ `O()`=実況主（ましろ）。個別コテはオブジェクト直書き。
  - 各post: `{no, name, trip, mail, date, body, op, img, slow}`。`mail:"sage"`で名前が青く。`slow:true`で“ここぞの一拍”。
  - `think`: 自分の番に**書き込み欄プレースホルダ**へ薄く出す「心の声」（誘導）。
  - `img:{src, fallback}`: 実画像。読込失敗時はコード生成SVG（`IMAGES[fallback]`）へ。
- レス番号は**通し番号（連番）**。本文の `>>N` は `seqMap` で連番へ自動変換。
- 実況主のトリップ/IDは**毎セッション ランダム生成**（`randTrip`/`randId`）。
- テンポ: `PACE` 定数（直前レスの文字数＝読む時間で待つ）。タップ/クリック/スペースで即送り。
- ペーシングの肝: 要所で「自分の番」になり、**コメントしないと進まない**（＝人間が速度を握る）。
- 自分の番はコメント欄を薄い黄色 `#fffbe2` でハイライト（スレ側の投稿ハイライトは無し）。

## 4. 新エピソードの作り方（シリーズ化）→「コードそのままコピーでOK」

**はい、`index.html` をコピーして中身を差し替えるだけ**でエンジンは再利用できます。
1. `index.html` を複製。
2. `SCRIPT` 配列を新しい話に差し替え（`posts` と `think` を書く。流れ・尺はここで決まる）。
3. `<title>` / `<meta name="description">` / スレタイ（`.thread-title`）/ 板名を変更。
4. 画像を差し替え（`img:{src}` の**半角ファイル名**に合わせる。例 `station.png`）。
   - ⚠️ 画像名は**半角推奨**。日本語名や ` (2)` 付きは配信で壊れやすい。差し替えは**同名で上書き**が安全。
5. 広告タグはそのまま流用可（or 新ユニット発行）。
- 描画・テンポ・連番・トリップ・広告・スマホ対応・SVGフォールバックは**全部そのまま動く**。

# 実装プロンプト: 裏サイト（2ch風 掲示板ハブ）

## 目的
単一ファイル型のスレ体験ゲーム（ささなき駅）を、複数スレを束ねる「掲示板（板）」
として提供し、回遊・滞在時間を伸ばす。エピソードをシリーズで追加できる構成にする。

## 画面構成
1. 板トップ（例 /ura/）= 2ch風スレ一覧
   - 板名ヘッダ、スレタイ一覧（各行: スレタイ / レス数 / 雰囲気の最終更新）

3. 戻る → 板トップ → 別スレへ（回遊）。各スレのクリア後に「他のスレも見る ▶」誘導。

## 技術方針（推奨A）
- エンジンを共通化: 現在の index.html の CSS/JS を engine.css / engine.js に外出しし、
  各スレは「データ＋画像＋engine読込」だけの薄いHTMLにする。
- 推奨構成A: **各スレを独立HTML**（/ura/sasanaki.html 等＝固有URL）。板トップは静的リンク集。
  → SEO・共有・回遊に最適（各スレが個別URLでインデックスされる）。
- 代替B: SPA（板トップ＋?thread=id でデータ動的読込）。UI一元管理だがSEOは各URL用意で補う。
